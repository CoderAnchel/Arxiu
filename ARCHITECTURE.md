# Architecture

> Living document. Updated as the system evolves.

## High-level

```

                                      │
                ┌─────────────────────┼─────────────────────┐
                │                     │                     │
        ┌───────▼──────┐      ┌───────▼──────┐      ┌───────▼──────┐
        │   frontend   │      │   backend    │      │    worker    │
        │ React (SPA)  │      │  FastAPI     │      │  ARQ + tasks │
        └──────────────┘      └───┬──────┬───┘      └───┬──────┬───┘

                              └───────┘             └───────┘
```

## Backend — FastAPI

- **Async throughout** (FastAPI + SQLAlchemy 2.0 async + asyncmy).
- **Layered**: API → service → CRUD → ORM. Permissions enforced at API _and_ service layers.
- **Pydantic v2** for request/response schemas; OpenAPI auto-published at `/api/v1/openapi.json` and consumed by the frontend code generator.
- **JWT RS256** access tokens (15 min) + refresh cookie (HttpOnly, Secure, SameSite=Strict, 7d, rotated).
- **ARQ** workers handle PDF generation, bulk email, Excel imports.
- **Structured JSON logs** via structlog → stdout → captured by Docker log driver.

## Frontend — React + Vite

- **TanStack Query** holds all server state with optimistic mutations (essential for the grade spreadsheet).
- **Zustand** holds UI state (theme, density, modal stack, toast queue, current curs acadèmic).
- **React Router v6** with lazy-loaded route chunks.
- **Themes** (`clar`, `fosc`, `editorial`) implemented via CSS variables on `[data-theme]`. Density via `[data-density]`. Both lifted from the design mockup in `Projects/Arxiu/`.

## Database — MySQL 8

- InnoDB, `utf8mb4_unicode_ci`, strict mode.
- **Soft delete** (`deleted_at`, `deleted_by_user_id`) on every entity. Permanent archive — no auto-purge.
- Migrations via Alembic, autogenerated then reviewed.

## Auth model

| Role | Login methods | Edit window |
|------|--------------|-------------|
| `admin` | DNI/email + password (admin-set or self-set), optional TOTP, optional Google OAuth | always |
| `professor` | DNI/email + admin-generated password, optional Google OAuth | per-RA notes when `assignacions_docents` matches AND avaluació is `docent`; any nota for tutored grup when avaluació is `junta` |

Password generation, regeneration, email-to-user (5-min retrieval window via Redis), and bulk CSV are admin-only operations under `/api/v1/admin/users/`.

## Permissions

Central pure-function module `app/core/permissions.py`. Each permission predicate takes `(user, target, avaluacio)` and returns a boolean. Predicates are unit-tested at 100% coverage and re-checked at the service layer (defence in depth).

## Audit

`audit_logs` is append-only. Every grade write, evaluation transition, import, export, mass email, password generation, and soft/hard delete records a row with before/after JSON snapshot, IP and user agent.

## Observability

- `/healthz` for liveness/readiness
- Prometheus exporter on `/metrics` (admin-restricted)
- Sentry for error tracking
- All logs JSON, correlation-IDs propagated via `structlog.contextvars`
